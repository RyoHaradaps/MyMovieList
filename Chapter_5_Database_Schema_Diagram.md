## Database Schema Summary

### **Core Tables:**
- **Profile:** User information and authentication
- **BaseContent:** Parent table for all content types
- **Genre:** Content categorization system
- **Watchlist:** User-curated content collections
- **Rating:** User-generated ratings and reviews

### **Content-Specific Tables:**
- **Movie:** Film-specific information and metadata
- **Series:** Television series information
- **AnimatedShow:** Animated content details
- **Comic:** Comic book and graphic novel data

### **Supporting Tables:**
- **Character:** Character information and relationships
- **Staff:** Creative personnel and their contributions

### **Key Relationships:**
- **One-to-Many:** Profile → Watchlist, Profile → Rating
- **Many-to-Many:** BaseContent ↔ Genre
- **One-to-One:** BaseContent → Content-specific tables
- **One-to-Many:** BaseContent → Character, BaseContent → Staff

### **Database Features:**
- **Normalized Design:** Efficient data storage and retrieval
- **Referential Integrity:** Maintains data consistency
- **Scalable Architecture:** Supports growth and new content types
- **Performance Optimization:** Indexed fields and optimized queries
